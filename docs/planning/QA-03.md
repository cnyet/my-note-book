## 第二轮：智能体工作流与业务规则

### 4. News Agent 的数据来源和展示逻辑？

- News Agent：自动爬取指定网站的AI资讯，并生成摘要，数据源：
  - 国际权威媒体: Google Research、MIT Tech Review、Hacker News
  - 中文优质资讯: 机器之心、36氪
  - 学术前沿: arXiv 最新论文
  - 社区讨论: Reddit 热议话题
- 不需要过滤关键词，你决定爬取的内容即可
- 每天展示10条资讯，格式：标题、摘要、链接、发布时间、来源
- 过往的资讯保存15天，用户可以查看历史AI资讯

### 5. Task Agent 的表单设计和数据结构？

- 问答表单是固定表单填写，你可以展示推荐的问题，生成的任务，在页面上已图表形式展示，并保存到对应的数据库中
- 填写表单后，AI分析后重新组织
- 生成的待办事项可以标记完成、修改、删除
- 如果今天的任务没完成，则标记未完成即可

### 6. Life Agent 的健康数据采集方式？

- 用户手动填写健康指标
- 健康指标字段：身高(150-200cm)、体重(40-120kg)、健康状况、运动频率、饮食偏好
- AI根据健康指标和历史数据，生成饮食和运动建议
- 健康数据需要加密存储，仅本地保存

### 7. Outfit Agent 的穿搭图片生成能力？

- 天气数据来源公共的天气API，生成的穿搭图片是AI生成，最好调用 Nano Banana API，或智谱大模型的API
- 日程安排来自用户输入，没有输入则AI推荐
- 根据天气和日程安排，推荐每日穿搭，并生成穿搭图片
- 穿搭图片是AI生成的虚拟模特穿搭

### 8. Review Agent 的复盘范围和用户偏好提取？

- Review Agent提取的"用户偏好"具体是：用户的身高、体重、健康状况、运动频率、饮食偏好、工作习惯、学习偏好等
- 偏好包括：工作习惯、兴趣标签、常用工具、时间管理模式等
- 提取的偏好存储在个人中心，其他Agent可以调用
- 如果AI提取的偏好不准确，用户可以手动修改

### 9. 5个智能体的工作流编排与依赖关系？

- News+Outfit → Task表单 → Life表单 → Review复盘，这个顺序是固定的
- 如果News Agent失败，Outfit Agent还会执行
- 如果Task表单未填写，Life Agent能正常运行
- 如果早上8:00定时执行失败（网络问题、API限流等），系统自动重试
- 用户可以跳过某个Agent，或单独重新执行某个Agent（比如重新生成穿搭建议）

### 10. LobeChat 调用5个智能体的具体场景？

- 用户在LobeChat对话时，通过特定指令（如"/news"）触发某个Agent
- LobeChat与5个Agent是独立运行的，不用相互调用
- LobeChat的对话历史（15天）与5个Agent的历史数据关联，可以查看历史AI资讯
