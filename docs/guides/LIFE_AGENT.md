# 生活秘书 (Life Secretary) 使用指南 🥗

## 1. 前言：健康即生产力

身体健康是开发者持续输出的基础。`LifeAgent` v2.0 通过结构化指标追踪与多模态输入，帮助大洪维持最佳状态。

---

## 2. 核心功能

- **📊 结构化指标 (Health Metrics)**：系统记录步数、睡眠时长、饮水量并存入数据库 `health_metrics` 表。
- **📸 多模态支持 (Vision)**：支持通过 `/api/life/vision` 接口上传照片，AI 自动识别食物热量或运动截图（需配置识别模型）。
- **🔄 闭环建议**：基于数据库中的历史指标趋势，动态调整今日的饮食与运动强度。

---

## 3. 使用场景

### 🟢 场景一：今日计划生成
点击 Dashboard 上的“生成生活建议”。
- 系统会检索数据库中最近 7 天的健康表现。
- 结合 LLM 生成包含饮食、运动、作息的完整方案。

---

## 4. 报告指标说明 (DB Fields)

| 板块 | 数据库字段 | 监控逻辑 |
| :--- | :--- | :--- |
| **步数** | `steps` | 连续 3 天低于 5000 步触发预警 |
| **睡眠** | `sleep_hours` | 结合昨日精力等级调整今日咖啡因摄入建议 |
| **饮水** | `water_intake` | 每日目标 2000ml 达成率追踪 |

---

## 5. 配置说明
目前的健康目标预设在 `backend/config/config.ini` 的 `[health]` 节点下。
