# 工作秘书 (Work Secretary) 使用指南 💼

## 1. 前言：从碎片到秩序

在繁杂的项目开发中，任务的优先级模糊是效率最大的敌人。`WorkAgent` 不仅是你的 TODO 列表生成器，更是能够跨越日期、自动继承遗留任务的“数字管家”。

---

## 2. 核心功能

- **记忆继承 (Legacy Sync)**：自动扫描前一天的 Markdown 日志，识别所有未打钩 (`[ ]`) 的任务并带入今日上下文。
- **🤖 AI 智能分级**：利用 LLM 根据任务描述自动标记高、中、低优先级。
- **⏱️ 时间块预估**：为每一项任务分配建议的起止时间，辅助番茄工作法实施。
- **🚀 多模态运行**：支持静默自动生成与深度交互定制。

---

## 3. 操作步骤

### 🟢 场景一：常规任务继承 (自动模式)
适用于没有新突发任务，仅需处理存量工作的情况。

1.  **执行命令**：
    ```bash
    cd backend
    python -m src.cli.main --step work
    ```
2.  **结果验证**：检查 `data/daily_logs/YYYY-MM-DD/work.md`，确认昨日未完成任务已出现在列表中。

### 🔵 场景二：新需求涌入 (交互模式)
适用于早晨例会后，有大量新任务需要录入的情况。

1.  **启动交互**：
    ```bash
    python -m src.cli.main --step work --interactive
    ```
2.  **输入内容**：根据提示输入新任务（如：“修复前端 404 刷新问题”、“下午三点参加架构评审”）。
3.  **智能规划**：AI 会将新输入的任务与继承的任务进行统筹，重新排序并分级。

---

## 4. 任务等级定义

系统使用 Emoji 进行直观区分：

| 等级 | 标识 | 处理策略 |
| :--- | :--- | :--- |
| **High** | 🚨 | **必须完成**。涉及 Deadline 或核心业务 Bug。 |
| **Medium** | ⚡ | **今日应完成**。若出现突发情况，可推迟至晚间。 |
| **Low** | 📝 | **琐事/优化**。利用碎片时间处理，无压力继承。 |

---

## 5. 最佳实践

- **保持原始记录**：在 Markdown 文件中手动标记任务状态（`[x]` 为完成），这是 `WorkAgent` 获取记忆的唯一途径。
- **描述清晰**：录入新任务时，尽量包含动词（如“重构”而非“模块”），有助于 AI 更准确地评估复杂度。
