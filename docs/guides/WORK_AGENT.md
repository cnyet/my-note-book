# 工作秘书 (Work Secretary) 使用指南

工作秘书 (`WorkAgent`) 是你的个人项目经理。它不仅帮助你规划当天的任务，还能确保你不会遗忘之前的待办事项。

## ✨ 功能介绍

*   **任务继承**：自动扫描昨日日志，提取未完成的任务（`[ ]` 标记）。
*   **智能优先级**：利用 AI 将任务自动分类为高、中、低优先级。
*   **时间建议**：为任务推荐合适的时间块。
*   **交互模式**：支持直接输入当天的额外任务或突发事项。

## 💻 使用方式

### 自动模式
仅基于昨日遗留任务生成今日计划：

```bash
cd backend
python -m src.cli.main --step work
```

### 交互模式 (推荐)
允许你在生成计划前输入今日的新任务：

```bash
cd backend
python -m src.cli.main --step work --interactive
```

系统会提示：
> Please enter any specific tasks or context for today...

## 📋 任务优先级说明

生成的待办清单使用以下 Emoji 进行优先级标记：

*   🚨 **High (高优先级)**: 必须在今日完成的核心任务，通常涉及截止日期或重要产出。
*   ⚡ **Medium (中优先级)**: 应该完成的任务，但如果时间紧迫可以推迟。
*   📝 **Low (低优先级)**: 琐事、行政工作或可以随时处理的任务。

## 🔄 继承机制

工作秘书的“记忆”机制非常简单有效：
1.  它会查找 `data/daily_logs/` 下**前一天**的文件夹。
2.  读取该文件夹下的 `work.md` (或整合日志)。
3.  提取所有包含 `[ ]` (未勾选) 的行。
4.  将这些内容作为上下文提供给 LLM。

**注意**：如果你手动在昨天的日志中将任务标记为 `[x]`，它们将不会被继承。

## 📄 输出示例

```markdown
# 今日工作计划

## 🚨 High Priority
- [ ] 完成 v2.0 架构文档编写 (建议: 09:30 - 11:30)
- [ ] 修复 API 鉴权 Bug (建议: 14:00 - 15:00)

## ⚡ Medium Priority
- [ ] 回复团队邮件 (建议: 13:30 - 14:00)

## 📝 Low Priority
- [ ] 整理桌面文件
```
