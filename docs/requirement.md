## 1. 项目概述

### 1.1 产品定位
Work-Agents 是一个面向极客群体的现代化AI工具聚合平台，提供AI助手服务、开发工具导航和实验性产品体验。平台采用极客美学设计，强调深色模式体验、流畅交互动效和技术社区氛围。

### 1.2 目标用户
- **主要用户**: 开发者、AI从业者、技术爱好者
- **管理员**: 内容运营人员、产品经理、系统管理员

### 1.3 核心价值
- 聚合优质AI工具和助手
- 提供极客风格的现代化体验
- 支持社区内容分享和互动

## 2. 功能需求

### 2.1 前台页面（5个）

#### 2.1.1 Home 首页
**功能定位**: 品牌展示与价值主张传递

**需求**:
- Hero 区域展示核心价值主张
- 品牌理念展示区（3列卡片）
- 热门 Agents 预览
- 响应式布局（桌面/平板/移动）

#### 2.1.2 Agents 页面
**功能定位**: AI助手集成与展示

**需求**:
- LobeChat iframe 集成
- 5个 AI 秘书展示和切换
- Agent 卡片展示（名称、图标、描述、状态）
- Tab 导航系统
- URL 同步（支持直接访问）

**愿景**: Agents 不是静态工具，而是有"个性"的AI伙伴
- 每个 Agent 可以有不同的"专长"（编程、写作、分析等）
- 未来愿景：Agent 之间可以协作和交流
- 用户可以根据需求组合使用不同 Agent

#### 2.1.3 Tools 页面
**功能定位**: 开发工具导航

**需求**:
- 工具集合分类展示
- 搜索功能
- 工具卡片（名称、图标、描述、外链）
- 分类和标签过滤

**愿景**: 工具不是静态列表，而是活的生态系统
- 工具之间可以有关联和依赖
- 展示工具的使用轨迹和影响力
- 让用户发现工具之间的协同效应

#### 2.1.4 Labs 页面
**功能定位**: 实验性产品展示

**需求**:
- 实验室产品列表
- 产品状态标识（Live/Beta/Prototype/Experimental）
- 产品详情展示（视频/图片、描述、技术栈）
- 在线用户计数器（实时）⏳ **Phase 2**

**愿景**: Labs = 创意试验田
- 最疯狂、最实验性的地方
- 展示"失败的项目"作为学习案例
- 用户可以提交自己的AI实验
- 实时的AI协作创作氛围

#### 2.1.5 Blog 页面
**功能定位**: 技术博客与内容营销

**需求**:
- 文章列表（分页、搜索、标签过滤）
- 文章详情（Markdown渲染、代码高亮、目录）
- 侧边栏（标签云、热门文章、归档）
- SEO 优化（OpenGraph、结构化数据）

**愿景**: Blog = 不是博客，是故事
- 不要传统技术博客格式，要真实的使用场景故事
- AI生成的艺术配图
- 用户投稿的"AI使用日记"
- 不仅是信息传递，更是"AI探索者联盟"的内部刊物

### 2.2 后台管理系统

#### 2.2.1 认证系统
**需求**:
- 用户注册和登录
- JWT 身份认证
- 角色权限控制（Admin/Editor）
- 速率限制（防止暴力破解）

#### 2.2.2 内容管理
**需求**:
- Agents CRUD
- 博客文章 CRUD（富文本编辑、标签管理）
- Tools CRUD
- Labs CRUD
- 媒体文件上传

#### 2.2.3 个人设置
**需求**:
- 个人信息编辑
- 密码修改

## 3. 技术约束

### 3.1 性能要求
- 首屏加载 < 1.5s
- API 响应时间 P95 < 200ms
- 支持 500+ 在线用户

### 3.2 兼容性
- Chrome/Edge、Firefox、Safari 最新2个主版本
- 响应式设计（Mobile/Tablet/Desktop/Wide）

### 3.3 安全要求
- 密码 bcrypt 加密
- HTTPS 强制
- 输入验证
- SQL 注入防护
- XSS 防护

### 3.4 数据库
- 支持关系型数据存储
- 轻量级解决方案优先
- 支持全文搜索

### 3.5 部署
- 支持 Docker 容器化部署
- 本地开发环境可运行

### 3.6 设计美学
**核心视觉追求**:
- **第一眼的"wow"感** - 让人印象深刻的第一印象
- **仔细看的"巧思"** - 有深度的细节设计
- **使用时的"流畅感"** - 顺滑的交互体验
- **离开时的"记住它"** - 独特的品牌记忆点

**视觉风格**:
- 暗黑色系，但不是死黑，带有渐变（紫色/蓝色基调）
- 赛博朋克元素，但不过度霓虹
- 极简主义，但不是"性冷淡"风格
- 科技感，但不冷冰冰

**动态效果**:
- 鼠标移动时的微妙视差
- 按钮 hover 时的粒子/光晕效果
- 页面滚动时的平滑过渡
- 打字机效果展示关键内容
- "呼吸感"的背景动画

**避免的**:
- 过于花哨的渐变
- 过多的动画导致分散注意力
- 过于"SaaS化"的设计语言
- 像教科书一样的刻板布局

### 3.7 情感诉求
**核心价值**:
- 这个网站不应该只是列出工具，而应该让用户"玩"起来
- 要有"灵魂"的东西，不是枯燥的产品说明书
- 体现极客精神：理解并掌控工具

**用户感受**:
- 极客归属感 - 为开发者、AI从业者、技术爱好者而设计
- 探索的乐趣 - 发现新工具的惊喜
- 创造的冲动 - 在 Labs 中实验、在 Blog 中分享
- 社区连接 - 不是孤立的工具目录，而是有温度的社区

## 4. UI/UX 规范

### 4.1 设计风格
- **参考网站**: checkmarx.dev、clawdbotai.co
- **主题**: 深色模式优先，支持浅色模式切换
- **风格**: 极客美学、现代感、动态效果

### 4.2 全局交互
- 主题切换（平滑过渡）
- 固定导航栏（玻璃态效果）
- 页面过渡动画
- 响应式布局

### 4.3 组件规范
- 一致的视觉语言
- 无障碍访问支持
- 交互反馈及时

```json
}
```
